# 元规范

## 六层结构

```text
.ai/
├── L0#Execution/     # 执行脚本、模板、工作流
├── L1#Overview/      # 项目入口和零容忍规则
├── L2#Index/         # 规范索引
├── L3#Standards/     # 当前唯一规范正文
├── L4#Changelog/     # 规范变更记录
└── L5#Knowledge/     # 可选知识沉淀
```

## 读取顺序

1. `L1#Overview/guide.md`
2. `L2#Index/toc.md`
3. 按需读取 `L3#Standards/standards/*.md`

## 写入顺序

1. 更新或新增 `L3#Standards/standards/*.md`
2. 更新 `L2#Index/toc.md`
3. 必要时更新 `L1#Overview/guide.md`
4. 在 `L4#Changelog/` 记录变更

## 禁止事项

- 禁止在 L2 写规范正文。
- 禁止在 L3 之外维护第二套当前规范。
- 禁止新增规范后不更新索引。
- 禁止让开发源码超过本项目大小规范。
