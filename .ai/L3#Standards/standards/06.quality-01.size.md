# 代码大小与拆分规范

## 纯化第一条规范

开发代码过程中，单个源码文件不得超过 1000 行。

这是本项目第一条强制纯化规范。任何新增功能、修复和重构都必须保持开发源码文件在 1000 行以内。

## 适用范围

- 适用：`src/`、`scripts/`、`.ai/` 中人工维护的源码和规范文件。
- 例外：`newapi-helper-suite.user.js` 是构建生成的油猴发布产物，不按开发源码行数限制判断。
- 例外：`ZERO/` 是外部参考副本，不按 NAHS 开发源码行数限制判断。

## 强制规则

- 禁止新增超过 1000 行的源码文件。
- 禁止继续向超过 1000 行的源码文件添加逻辑。
- 文件接近 900 行时必须优先拆分。
- 拆分后仍由 `scripts/build.mjs` 拼接发布产物。

## 当前拆分方式

- UI 源码：`src/ui.part*.js`
- 核心逻辑：`src/core.part*.js`
- 样式源码：`src/styles.part*.css`
- 运行时入口：`src/runtime/*.js`

## 检查命令

```bash
wc -l src/*.js src/*.css src/runtime/*.js scripts/*.mjs
npm run check
git diff --check
```
