# NAHS - NewAPI Helper Suite

NAHS（NewAPI Helper Suite）是一个面向 NewAPI 管理后台的油猴脚本辅助工具箱。

当前版本的第一个工具模块是“批量添加渠道 / key 池作业”：可以批量入库 provider key，按作业参数创建渠道，并持续监控存活数量，低于阈值时自动从 key 池补充新渠道。脚本支持当前浏览器登录态模式，也支持填写远端 NewAPI 地址和用户密钥后通过 API 管理。

## 安装与更新

推荐使用 Tampermonkey / Violentmonkey 的“从 URL 安装”方式，地址填写：

```text
https://raw.githubusercontent.com/LickJCR/NAHS/refs/heads/main/newapi-helper-suite.user.js
```

这是项目的单文件发布产物。安装后，脚本里的 `@updateURL` 也指向同一个地址，后续更新会继续从这里检查和拉取。

脚本会匹配 `http(s)://*/*`，但运行时只在疑似 NewAPI 页面挂载按钮。打开任意 NewAPI 页面即可，不要求必须先进入渠道页面。

不建议手动复制 `src/` 目录里的源码到油猴；`src/` 只用于开发维护。需要手动复制时，也应该复制生成后的 `newapi-helper-suite.user.js`。

## 当前功能

- 支持 NewAPI v0.13.2 和 v1.x 登录态。
- 打开工作台后先选择“当前浏览器渠道 key 批量添加”或“远端 NewAPI 管理”。
- 直接调用 `/api/channel` 创建渠道。
- 远端模式支持保存最多 5 个 NewAPI 台子，每个台子独立保存 key 池、作业、日志和列表缓存。
- 每个远端台子下包含 `批量添加`、`渠道列表`、`日志列表`、`用户列表` 四个子标签。
- 远端 `渠道列表`、`日志列表`、`用户列表` 按 NewAPI 默认列表的列顺序、列头和主要字段模拟展示。
- 远端连接测试会显示连接状态、站点信息、账号信息和 API 检查结果。
- 支持 key 池入库、去重、状态统计和工作记录持久化。
- 支持创建作业后持续监控渠道状态。
- 支持保活数量、低于阈值、补充数量、监控间隔等策略参数。
- 支持按渠道类型读取分组、样板渠道、模型和模型映射。
- 支持自定义名称组合：固定文本、顺序数字、顺序字母、随机码、时间戳、日期、key 前缀。
- 支持导入 / 导出工作记录，迁移后继续执行。

## 规划

这个项目会逐步扩展为 NewAPI 辅助工具箱，而不是只做批量添加。

- 日志错误监控。
- 关键事件声音提醒。
- 渠道异常报警。
- 额度 / 费用 / 存活时间统计看板。
- 作业历史分析和本地日志导出。
- 更多 NewAPI 管理后台辅助操作。

## 使用说明

1. 按上面的 URL 安装脚本，并确认脚本管理器已启用。
2. 在已登录的 NewAPI 管理后台打开任意页面。
3. 点击页面上的 `批量渠道 / NACP` 浮动按钮打开工作台；按钮可以拖动调整位置。
4. 选择工作模式：`当前浏览器渠道 key 批量添加` 会使用当前页面登录态和当前站点 localStorage；`远端 NewAPI 管理` 会进入远端台子管理。
5. 远端模式下填写台子名称、NewAPI 地址、User ID、User 密钥和密钥发送方式，然后点击“保存并新增台子”。最多保存 5 个台子。
6. 不同 NewAPI 部署如有差异，可先点“测试连接”确认。测试结果会显示连接状态、站点名称 / 版本、账号 ID / 名称 / 分组 / 额度，以及站点状态、分组 API、渠道 API、账号信息 API 的检查结果。
7. NewAPI 地址可以粘贴控制台页面地址，例如 `/console/personal`；保存或测试时会自动归一为站点根地址。
8. 每个台子都有独立的 `批量添加`、`渠道列表`、`日志列表`、`用户列表` 子标签。切换台子时，key 池、作业、日志和列表缓存也会随台子切换。
9. 在左侧 `key 库` 粘贴 provider key，点击“添加入库”。支持逐行、逗号分隔、JSON 数组、`key=value`、带序号或引号的列表；key 会先进入当前工作记录的本地 key 池。
10. 在中栏点击“创建作业参数”，打开右侧作业配置区。
11. 在右侧配置渠道类型、分组、模型、模型映射、名称组合、优先级、权重、标签、备注和高级 JSON 字段。也可以点击刷新分组 / 样板渠道，从当前或远端 NewAPI 站点读取已有配置作为参考。
12. 配置策略：`保活` 是目标存活数量，`低于` 是触发补货阈值，`添加` 是每次补充数量，`间隔` 是自动监控周期。
13. 点击“刷新预览”确认渠道名称和 key 解析结果，必要时可复制首条 payload 检查提交内容。
14. 点击“保存创建作业”。脚本会先按保活数量创建首批渠道，然后定时刷新渠道状态；当存活数量低于阈值时，会从 key 池继续补充。
15. 作业创建后，可在中栏查看作业统计、批次记录和作业日志；也可以暂停 / 开启作业、手动刷新状态，或修改运行策略后点击“应用策略”。
16. 远端模式下，`渠道列表`、`日志列表`、`用户列表` 会通过远端 API 动态读取，并按 NewAPI 默认列表的列顺序、列头和主要字段展示。
17. 顶栏的“导出工作”会导出完整 key 池、作业和日志；“导入工作”可在其他浏览器或站点恢复；“重置”会清空当前工作记录。远端模式下，“重置”还会清空已保存的远端台子、连接状态和远端列表缓存。

key 池为空时也可以先创建作业，后续继续入库 key 后，作业会在自动监控时补货。

## 开发

仓库使用多文件源码、单文件发布的方式：

- `src/` 是开发源码。
- `src/*.part.js` 和 `src/*.part.css` 是拆分后的开发源码，单文件保持在 1000 行以内。
- `.ai/` 是本项目开发规范，参考 `ZERO/.ai` 建立。
- `ZERO/` 是 `LickJCR/ZERO` 的本地参考副本。
- `scripts/build.mjs` 会把源码拼成 `newapi-helper-suite.user.js`。
- `newapi-helper-suite.user.js` 是油猴安装和自动更新使用的发布产物，不建议直接编辑。

修改源码后运行：

```bash
npm run build
npm run check
```

## 安全说明

- API key 不会写入脚本仓库。
- 工作记录默认保存在当前站点浏览器 localStorage，刷新页面不会丢失。
- 远端 NewAPI 地址、User ID 和 User 密钥也保存在当前站点浏览器 localStorage；不要在不可信浏览器中保存远端密钥。
- 浏览器 localStorage 按 origin 隔离：不同协议 / 域名 / 端口的 NewAPI 站点会分别保存；同一个 origin 下的不同路径会共用同一份工作记录。
- 导出工作记录时请自行保管文件，避免泄露 key 信息。
